Building my first portfolio website was an exciting journey that combined creativity with technical skills. As a computer science student at FAST, I wanted to create something that showcases my abilities while being functional and visually appealing.

Why Build a Portfolio Website?

In today's digital age, having an online presence is crucial for any aspiring software engineer. A portfolio website serves multiple purposes:

  • Showcases your projects and skills
  • Demonstrates your coding abilities
  • Provides professional credibility
  • Serves as a learning project itself

Planning the Structure

Before writing any code, I spent time planning the website structure:


                    1. Hero Section with introduction
                    2. About Me section
                    3. Education and Skills
                    4. Project Gallery
                    5. Blog Section
                    6. Contact Form

Technical Stack

I kept it simple for my first version:

"Start simple, then iterate. Don't try to build the perfect website on your first attempt."

HTML5: For semantic structure

CSS3: For styling and responsive design

Google Fonts: For typography (Cause & Intel One Mono)

Flexbox & Grid: For layout management

Challenges Faced

Some challenges I encountered:

  1. Responsive Design: Making the site look good on all devices
  2. Cross-browser Compatibility: Ensuring consistent experience
  3. Performance Optimization: Keeping load times fast
  4. Accessibility: Making the site usable for everyone

Lessons Learned

Through this project, I learned:

  • The importance of mobile-first design
  • How to use CSS Grid effectively
  • The value of semantic HTML
  • Basic SEO principles
  • Version control with Git

Next Steps

Planned improvements for version 2.0:

1. Add dark mode toggle
2. Implement search functionality
3. Add more interactive elements
4. Integrate a CMS for blog posts
5. Add project filtering system
